汇编语言学习(1)

3月 4, 2020

1.1 机器语言

机器语言就是机器指令的集合

1.2 汇编语言的产生

汇编语言的主体是汇编指令

举个例子
机器指令: 1000100111011000
操作: 寄存器BX的内容送到AX中
汇编指令: MOV AX,BX

寄存器：简单地讲是cpu中可以存储数据的器件，一个cpu有多个寄存器。
AX 是其中一个寄存器的代号，BX 是另一个寄存器的代号。

1.3 汇编语言的组成

  1. 汇编指令 (机器码的助记符) 核心
  2. 伪指令 (有编译器执行)
  3. 其他符号 (有编译器识别)

1.4 存储器

计算机大部分部件都存在存储器
指令和书记在存储器中存放，也就是平常所说的内存

1.5 指令和数据

在内存或磁盘中，指令和数据是没有任何区别的，都是二进制信息。
举个例子
1000100111011000 -> 89D8H(数据)
1000100111011000 -> MOV AX,BX(程序)

1.6 存储单元

存储器被划分为若干个存储单，每个存储单元从0开始顺序编号
举个例子
一个存储器有128个存储单元，则编号从0~127.
存储单位：
1KB = 1024B
1MB = 1024KB
1GB = 1024MB
1Tb = 1024GB

1.7 CPU对存储器的读写

CPU要想进行数据的读写，必须和外部期间进行三类信息的交互：

  1. 存储单元的地址 (地址信息)
  2. 器件的选择,读或写 (控制信息)
  3. 读或写的数据 (数据信息)

计算机连接CPU和其他芯片的导线，通常称为总线。
逻辑上可划分为：

  1. 地址总线
  2. 数据总线
  3. 控制总线

对于8086CPU，下面的机器码能够完成从3号单元读数据：
机器码：101000000000001100000000
含义：从3号单元读取数据送入寄存器AX
CPU接受到这条机器码后将完成上面所述的读写工作

1.8 地址总线

cpu是通过地址从总线来指定存储单元。
CPU有N根地址总线，则可以说这个cpu的地址总线的宽度位N。
这样的CPU最多可以寻找2的N次方个内存单元。
1个地址总线可以索引1个内存单元，1个内存单元就是1个字节(1Byte) = 8位(8bit)

地址总线上能传送多少个不同的信息，CPU就可以对多少个存储单元进行寻址。
32位和64位的区别就在于这。

举个例子：
要想说你使用的是64位计算机，你需要满足以下条件：

  1. 拥有64位CPU
  2. 拥有64位操作系统
  3. 拥有64位软件

1.9 数据总线

CPU与内存或其他器件之间的数据传送是通过数据总线来进行的。
数据总线的宽度决定了CPU和外界的数据传送的速度。
8088 -> 8位
8086 -> 16位

1.10 控制总线

cpu对外部器件的控制是通过控制总线来进行的。在这里控制总线是个总称，控制总线是一些不同控制线的集合。
有多少根，就提供多少钟控制。

1.11 内存地址空间

一个CPU的地址线宽度为10，那么可以寻址1024个内存单元，这1024个可寻到的内存单元就构成这个CPU的内存地址空间。

1.12 主板

每一台计算机中，都有一个主板，主板上有核心器件和一些重要器件。
这些器件通过总线相连。

1.13 接口卡

计算机系统中，所以可用程序控制其工作的设备，必须受到cpu的控制。
CPU对外部设备不能直接控制。

1.14 给雷存储器芯片

  1. 从读写属性分为 随机存储器(RAM) 和 只读存储器(ROM)
  2. 从功能和连接上分类 随机存储器RAM 装有BISOS的ROM 接口卡上的RAM

1.15 内存地址空间

系统不同会分配不同地址空间
最终运行程序的是CPU，所以我们用汇编编程的时候，必须要从CPU角度考虑问题。
